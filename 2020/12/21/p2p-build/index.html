p2p-build

p2p分发文件搭建测试

实验条件：

硬件设备:

公网ip的tracker服务器,两台客户端机器(可以使用虚拟机代替)

软件：

客户端:

qBittorrent(不建议使用公共代理访问网页，会被禁止访问)

tracker服务器软件:

bittorrent-tracker(本次测试用于Ubuntu20.04LTS x86_64)

BitComet Tracker(本次测试用于windows server 2008 x86_64)

qBittorrent的高级选中打开内置tracker端口

1.qBittoren制作种子torrent文件

注意：

种子客户端即上传宿主机必须是公网ip，否则无法上传文件，初步判定为nat技术缺陷，软件无法解决。

步骤:

  1. 打开qBittorrent主界面

  2. 选择”工具”, 点击生成Torrent,跳出如下界面

  3. 选择文件或者选择文件夹后，选择完成后开始做种，否则需要手动做种，手动做种需要添加相应的torrent文件，如何对应的位置不存在文件，则会重新下载文件。添加tracker服务的链接，图片中的链接是内网中测试时搭建的。

  4. 成功做种会生成torrent文件，同时在下载项显示做种

    2.搭建tracker服务器

    BitComet Tracker是免费许可软件，不开源，最后一次更新是2007年，比较适合于老版本的服务器，比如windows server 2008。

bittorrent-tracker是开源免费软件，要求nodejs版本>=10(windows server 2008无法安装高版本的nodejs), 版本过低，能运行但是会出现错误。比较适用于新版本的系统，而且能进行二次开发，优先推荐

windows server 2008(公网ip)

从官方上下载zip文件后，上传到服务器并解压, 双击BitCometTracker.exe，弹出下框

打开软件之后，状态是停止的的状态，需要点击”run”

这样就行了，如果需要修改配置端口，需要点击“config”


配置windows防火墙，允许外部访问8080端口，防火墙高级选项中添加8080端口入站规则。本人出于测试方便直接关闭防火墙。
tracker链接：http://ip地址:8080/announce

ubunut20.04(内网ip)

安装npm

安装bittorrent-tracker

如出现npm无法找到源,请检查拼写是否存在错误，bittorrent-tracker很容易拼写错误而且发现错误，最好复制粘贴

启动bittorrent-tracker

输出如下：

则启动成功
ubuntu配置防火墙，允许外部访问8000端口

本人出于测试方便直接关闭防火墙。

解决内网种子用户无法上传文件设想方案(本人尚未完全进行测试，纯属参考)：

可考虑使用内网穿透端口映射frp+vpn,二者缺一不可。

frp内网穿透经本人测试后，发现只能外网访问内网机器,内网访问tracker服务器依旧是nat转换后的ip地址，不可用，所以需要使用vpn访问tracker服务器记录内网BT端口内网穿透映射后的地址。后期开发，可考虑整合qBittorrent和frp实现内网种子客户端作为上传宿主机，同时二者都是开源软件
